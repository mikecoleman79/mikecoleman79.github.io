Age Verification Integration for WooCommerce

Client: Popcorn Web Design · Platform: WordPress & WooCommerce

Project Overview

This project involved implementing a compliant age verification flow for selected age-restricted products on an existing WooCommerce site. The solution required integration with a third-party age verification service (OneID) while ensuring minimal impact on the standard checkout experience.

The site already had an established customer base, meaning the solution needed to be robust, legally compliant, and carefully integrated into existing cart and checkout logic.

The Challenge

  • WooCommerce does not provide native age verification functionality
  • Only specific products required verification
  • Legal and compliance risks if verification failed
  • No sensitive user data could be stored locally
  • Checkout flow needed to remain smooth for non-restricted products

Technical Solution

1. Setup & Authentication

OneID API credentials were configured and OAuth authentication implemented. Initial API connectivity was verified using Postman and cURL before development began.

2. API Integration

A dedicated API handler was created to manage verification requests and responses. Verification states were normalised to ensure consistent handling across the application.

  • Secure token handling
  • Graceful handling of API errors and timeouts
  • Clear verified / not-verified states

3. WooCommerce Cart & Checkout Logic

A custom product meta field (is_age_restricted) was introduced to flag products requiring verification.

  • Cart scanned for restricted items
  • Verification status stored in session
  • Checkout blocked if verification not completed
  • Orders prevented via woocommerce_checkout_process

4. UI / UX Considerations

Clear messaging was added to inform users when age verification was required. This was implemented using a modal or redirect flow to avoid disrupting the checkout experience unnecessarily.

5. Testing & Edge Cases

  • Mixed carts (restricted + unrestricted products)
  • Failed or abandoned verification attempts
  • API downtime and fallback behaviour
  • Session persistence across page reloads

6. Deployment

The integration was migrated from sandbox to production with additional logging enabled to monitor live usage and identify any potential issues early.

Outcome

  • Legally compliant age-restricted checkout flow
  • No impact on standard product purchases
  • Secure, stateless verification handling
  • Maintainable, plugin-based architecture

Technology Stack

WordPress WooCommerce PHP JavaScript OneID API OAuth

Key Takeaways

This project highlights a senior-level approach to integrating third-party services into WooCommerce, balancing compliance, performance, user experience, and long-term maintainability.
