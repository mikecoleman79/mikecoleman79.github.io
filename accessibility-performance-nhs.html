Accessibility & Performance Remediation for a Public-Sector Platform

Role: Senior WordPress Developer
Agency: Digital agency (NDA / confidential)
Client: NHS sub-organisation (public sector, NDA)

WordPress WCAG 2.1 Accessibility Performance Gravity Forms

Overview

I led the remediation of accessibility and performance issues across a live public-sector digital platform operated by an NHS organisation.

The work followed a formal external accessibility audit conducted under the Public Sector Bodies (Websites and Mobile Applications) Accessibility Regulations 2018, requiring full compliance with WCAG 2.1 AA. My role involved translating audit findings into practical technical improvements while ensuring changes could be safely deployed to a live, high-visibility service.

The Audit

An independent audit identified a wide range of accessibility failures across multiple page templates, navigation components, forms and embedded content

Issues were identified through a combination of automated testing tools and manual assessment, covering keyboard navigation, focus visibility, contrast, reflow at high zoom levels and semantic markup.

Key Challenges

  • Keyboard-only users unable to access core navigation and menus
  • Loss of visible focus in carousels, buttons and form elements
  • Hover-only interactions excluding keyboard and touch users
  • Cookie banners obscuring focused elements
  • Layouts breaking at 200–400% zoom
  • Inaccessible markup generated by third-party plugins
  • Live production site with ongoing public usage

Approach & Decisions

Keyboard Navigation & Focus Management

  • Refactored navigation menus to be fully keyboard accessible
  • Ensured logical focus order across interactive elements
  • Added clear, consistent focus styles throughout the site
  • Resolved focus loss within carousels and dynamic components

Hover, Focus & Interactive Content

  • Removed reliance on hover-only interactions
  • Ensured content triggered on hover was also available via keyboard
  • Implemented dismissible patterns for hover/focus content

Zoom, Reflow & Responsive Behaviour

  • Tested layouts at 200% and 400% zoom levels
  • Adjusted layouts to reflow correctly without horizontal scrolling
  • Maintained usability across desktop and mobile breakpoints

Contrast & Visual Accessibility

  • Improved colour contrast for text, links and buttons
  • Balanced brand constraints with WCAG requirements
  • Used automated tooling where appropriate, with manual fixes where required

Forms & Third-Party Remediation

  • Identified invalid markup produced by Gravity Forms
  • Resolved empty lists, duplicate IDs and invalid ARIA references
  • Ensured file uploads and validation messages were accessible
  • Applied fixes globally to prevent regressions

Platform-Wide Improvements

  • Fixed header, footer and cookie banner issues once, site-wide
  • Reduced duplication by addressing root causes
  • Ensured future pages inherited accessibility improvements

Performance Considerations

Several accessibility improvements also resulted in performance benefits, including reduced DOM complexity, simplified JavaScript behaviour and fewer layout shifts at high zoom levels.

These changes improved reliability for keyboard users while also contributing to faster interaction times and a more stable user experience.

Outcome

  • WCAG 2.1 issues resolved across core templates and components
  • Improved experience for keyboard, screen reader and low-vision users
  • Reduced compliance risk for a public-sector platform
  • Sustainable, platform-level accessibility improvements
